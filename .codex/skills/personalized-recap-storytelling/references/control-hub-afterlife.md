# Control Hub And Afterlife

Wrapped-like systems are no longer just linear stories.

## Control

Define:
- replay
- rewind
- skip
- resume
- pace or speed adjustment
- re-entry after exit

If the run is fast, users need a recovery path.

## Hub

A good recap usually needs a persistent home where users can:
- revisit stories
- find messages or clips
- collect derived artifacts
- rewatch or reshare key moments

## Afterlife

Choose what persists after the run:
- playlist
- saved summary
- badge or role
- benchmark
- recommendations
- links into the main product

## Forward Bridge

The best recap points forward, not only backward.
Give the user a next action that feels earned by the recap:
- replay
- explore more
- compare
- save
- improve

## Product Rule

Do not treat the linear run as the whole product.
Design the run, the hub, and the post-run artifact together.
