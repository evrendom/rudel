# Tracked Vs Missing

Use this split every time you propose a new beat.

## Tracked Directly

- favorite model and source split
- sessions, active days, spend, duration, and total tokens
- average session duration
- success rate and success rate trend
- top skills, slash commands, and subagents
- distinct projects, repositories touched, and total commits
- session archetype, success score, plan mode usage, commit flag, interaction count, and error count

## Derivable With Existing Data

- model distribution
- dominant session archetype
- commit-session rate
- top project by tokens or sessions
- peak day and longest streak
- richer workflow or tooling identity cards
- share-card summaries based on existing mapped fields

## Missing Or Not Persisted

- timezone or locale on user or org
- walk-in onboarding state
- final user-level wrapped archetype persistence
- model context capacity metadata for context-window percentage stories
- canonical model family taxonomy

## Practical Rule

If a metric is only derivable, say so and name the query or adapter work needed.
If it is missing, do not present it as a copy problem.
