# Delight And Visual Language

Delight in the article is deliberate and controlled.

## Delight

- Add warmth intentionally.
- Make difficult actions feel welcoming.
- Use memorable accents sparingly but deliberately.
- Keep the tone human, soft, and considered.

## Visual language from the references

- bright or light backgrounds
- large open whitespace
- rounded geometry everywhere
- bold but friendly headlines
- quiet secondary copy
- wide pill CTAs
- low visual noise
- one focal object or one focal action per screen

## Screen-level cues

- Start screen: playful top illustration field, strong headline, two simple CTA pills.
- Wallet ready screen: sparse layout, focal wallet card, one main CTA.
- Email and code screens: light shell, compact top progress, restrained helper affordances, direct task framing.

## Adaptation guidance

- Move away from the current dark flight-board aesthetic unless explicitly asked to retain it.
- Prefer soft hierarchy over dense theatrical framing.
- Use rounded cards and product objects as anchors instead of heavy dashboard chrome.
- Keep the route feeling like a mobile wallet product, not a control room.

## Warning

- Do not confuse delight with visual overload.
- Do not add decorative complexity that fights simplicity or continuity.
